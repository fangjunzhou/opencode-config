# OpenCode Config

My OpenCode prompts and configurations for agentic coding workflows.

## Overview

This repository contains OpenCode agent definitions and configuration files that define how AI coding agents operate across different language and project ecosystems. Configurations are organized by variant (base, Python, C++, etc.) and publicly accessible for agents to consume.

## Quick Start

### Install Base Configuration

```bash
curl -fsSL https://fangjun.github.io/opencode-config/distributions/opencode-base.tar.gz | tar xz
```

### Install Python-Specific Configuration

```bash
curl -fsSL https://fangjun.github.io/opencode-config/distributions/opencode-python.tar.gz | tar xz
```

### Browse Configuration

Visit: **https://fangjun.github.io/opencode-config/**

## How It Works

### Architecture

1. **shared-config/** - Single source of truth for all variants
2. **language-specific/** (python/, cpp/, etc.) - Override/extend shared configs
3. **.opencode/** - Generated by merging shared + variant configs
4. **GitHub Pages** - Hosts file browser + downloadable distributions

### Merge Strategy

When running `./scripts/setup-opencode.sh python`:

```
shared-config/*  ──┐
                   ├──> Copy to .opencode/
                   │    (base configs)
                   │
python/*         ──┘    Overlay on top
                        (python-specific overrides)
```

Result: `.opencode/` contains base + Python-specific configs

### Workflow

```mermaid
Developer Push
    ↓
Build All Variants
    ├── Base variant
    ├── Python variant
    └── Create distributions
    ↓
Generate Index Page
    ├── List available downloads
    ├── Show checksums
    └── Create .tar.gz files
    ↓
Deploy to GitHub Pages
    ├── File browser (index.html)
    ├── Distributions folder
    ├── shared-config/ (browseable)
    └── .opencode/ (browseable)
    ↓
Agents Can Download & Access
    ├── HTTP file access
    ├── Plaintext rendering
    └── No authentication
```

## Setup Instructions

### For Development

1. **Edit shared configs:**
   ```bash
   vi shared-config/agents/autobuild.md
   ```

2. **Edit variant configs:**
   ```bash
   vi python/commands/add-lib.md
   ```

3. **Regenerate .opencode/:**
   ```bash
   ./scripts/setup-opencode.sh python
   ```

4. **Test locally:**
   - Verify `.opencode/` has both shared + variant configs
   - Commit all changes

### Building Distributions Locally

```bash
# Build single variant
./scripts/build-distributions.sh base

# Build all variants
./scripts/build-distributions.sh all

# Generate index page
python3 scripts/generate_distribution_index.py distributions/
```

Output: `distributions/opencode-{variant}.tar.gz` files

## Agent Definitions

### Shared Agents

Located in `shared-config/agents/`:

- **autobuild.md** - End-to-end implementation agent that executes, validates, and researches changes
- **autoplan.md** - Research-heavy planning agent for complex task analysis

### Python-Specific Agents

Located in `python/agents/` (override shared if needed):

- Create Python-specific versions if agents need special behavior

### Format

Each agent is a Markdown file with YAML frontmatter:

```markdown
---
description: Agent description
mode: subagent | primary
temperature: 0.0-1.0
permission:
  read: allow|deny
  edit: allow|deny
  bash: allow|deny
  # ... more permissions
---

Agent instructions and guidelines...
```

## CI/CD Automation

### Workflows

1. **deploy-directory-listing.yml** - Generates and deploys file browser
   - Triggers on pushes affecting .opencode/, shared-config/, etc.
   - Generates index.html with complete file tree
   - Deploys to GitHub Pages root

2. **distribute-variants.yml** - Builds and deploys distributions
   - Triggers on pushes affecting shared-config/, scripts/, etc.
   - Builds all variants in parallel
   - Creates checksums and index page
   - Deploys to GitHub Pages /distributions/ folder

### Manual Trigger

```bash
gh workflow run deploy-directory-listing.yml
gh workflow run distribute-variants.yml
```

## Adding New Variants

To add a C++ variant:

1. **Create directory:**
   ```bash
   mkdir -p cpp/{agents,commands}
   ```

2. **Add C++-specific configs:**
   ```bash
   cp python/README.md cpp/README.md  # As template
   # Create cpp/commands/add-lib.md, etc.
   ```

3. **Update workflow:**
   - Edit `.github/workflows/distribute-variants.yml`
   - Add `cpp` to the matrix

4. **Test locally:**
   ```bash
   ./scripts/setup-opencode.sh cpp
   ./scripts/build-distributions.sh cpp
   ```

5. **Commit and push:**
   ```bash
   git add cpp/ .github/workflows/distribute-variants.yml
   git commit -m "Add C++ variant"
   git push
   ```

Workflows will automatically generate `opencode-cpp.tar.gz`

## Development Guidelines

Refer to [AGENTS.md](AGENTS.md) for detailed guidelines when making changes to agent definitions and configurations.

### Key Rules

- **Do not edit .opencode/ directly** - It's generated
- **Edit shared-config/ or variant/** instead
- **Run setup script** after making changes: `./scripts/setup-opencode.sh {variant}`
- **Test locally** before committing
- **Commit both source and generated files** (.opencode/)
